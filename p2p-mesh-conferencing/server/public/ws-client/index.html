Events that clients can send to the server

CLIENT
send-message
Send a text message to the room
▼
Message Schema:
{ "type": "send-message", "data": "string (1-1000 characters)" }

Validation Rules:

  • Message cannot be empty
  • Maximum length: 1000 characters
  • Message will be trimmed

Events that the server sends to clients

SERVER
connected
Connection confirmation with user data
▼
Response Schema:
{ "type": "connected", "data": { "id": "string", "name": "string", "roomId": "string", "connected": boolean, "joinedCall": boolean } }
SERVER
pong
Response to ping message
▼
Response Schema:
{ "type": "pong" }
SERVER
message-sent
Confirmation that message was sent
▼
Response Schema:
{ "type": "message-sent", "data": { "id": "string", "roomId": "string", "userId": "string", "content": "string" } }
SERVER
message-received
Broadcast when a message is received in the room
▼
Response Schema:
{ "type": "message-received", "data": { "id": "string", "roomId": "string", "userId": "string", "content": "string" } }
SERVER
user-connected
Broadcast when a user connects to the room
▼
Response Schema:
{ "type": "user-connected", "data": { "id": "string", "name": "string", "roomId": "string", "connected": boolean, "joinedCall": boolean } }
SERVER
user-disconnected
Broadcast when a user disconnects from the room
▼
Response Schema:
{ "type": "user-disconnected", "data": "string" // User ID }
SERVER
joined-conference-as-publisher
Response when successfully joined conference as publisher
▼
Response Schema:
{ "type": "joined-conference-as-publisher", "data": { "room": number, "feed": StandardizedPublisher, "publishers": StandardizedPublisher[] } }

StandardizedPublisher Object:

  • id: number (feed ID)
  • feedType: "camera" | "screenshare"
  • userId: string
  • audio: boolean
  • video: boolean
  • talking: boolean
  • publisher: boolean
  • handRaised: boolean
SERVER
receive-answer-for-publishing
WebRTC answer from server for publishing stream
▼
Response Schema:
{ "type": "receive-answer-for-publishing", "data": { "configured": "ok", "audio_codec": "string", "video_codec": "string", "streams": array, "jsep": { "type": "answer", "sdp": "string" } } }
SERVER
subscribed-to-user-feed
WebRTC offer from server when subscribed to a user feed
▼
Response Schema:
{ "type": "subscribed-to-user-feed", "data": { "room": number, "streams": [{ "mid": "string", "type": "audio" | "video" | "data", "feed_id": number }], "jsep": { "type": "offer", "sdp": "string" } } }
SERVER
publisher-list
List of active publishers in the room
▼
Response Schema:
{ "type": "publisher-list", "data": any[] }

Typical Publisher Object Structure:

  • { "id": number, "display": "string", "audio_codec": "string", "video_codec": "string" }
  • Note: Actual structure depends on Janus VideoRoom plugin response
SERVER
media-stream-toggled
Confirmation that media stream was toggled
▼
Response Schema:
{ "type": "media-stream-toggled", "data": { "video": boolean, "audio": boolean } }

Note:

  • This event confirms the media stream toggle was processed
  • Sent in response to toggle-media-stream client message
SERVER
feed-unpublished
Confirmation that feed was unpublished
▼
Response Schema:
{ "type": "feed-unpublished", "data": { "feedId": number } }

Note:

  • This event confirms the feed unpublish was processed
  • Sent in response to unpublish-feed client message
  • The feed is now removed from the conference
SERVER
publisher-unpublished-feed
Broadcast when a publisher unpublishes a feed
▼
Response Schema:
{ "type": "publisher-unpublished-feed", "data": { "userId": "string", "feedId": number } }

Note:

  • This event is broadcast to all users in the call
  • Indicates that a publisher has stopped sharing a specific feed
  • Subscribers should handle feed removal from UI
SERVER
screenshot-received
Broadcast when a user takes a screenshot during the call
▼
Response Schema:
{ "type": "screenshot-received", "data": { "userId": "string", "userName": "string" } }

Note:

  • This event is broadcast only to users in the call
  • Triggered when a user sends a screenshot notification
  • Contains the user ID and name of who took the screenshot
  • Can be used to show notifications or update UI
SERVER
reaction-received
Broadcast when a user sends a reaction in the call
▼
Response Schema:
{ "type": "reaction-received", "data": { "userId": "string", "userName": "string", "reaction": "string" } }

Note:

  • This event is broadcast only to users in the call
  • Contains the user ID, name, and the reaction sent
  • Reactions are limited to 50 characters
  • Can be used to display reactions in the UI temporarily
SERVER
hand-raised
Broadcast when a user raises their hand during the call
▼
Response Schema:
{ "type": "hand-raised-by-user", "data": { "userId": "string", "feedId": number } }

Note:

  • This event is broadcast only to users in the call
  • Contains the user ID and feed ID of who raised their hand
  • Can be used to show a hand raised indicator in the UI
  • Triggered when a user sends a raise-hand message
  • feedId identifies which specific media handle raised the hand
SERVER
hand-lowered
Broadcast when a user lowers their raised hand during the call
▼
Response Schema:
{ "type": "hand-lowered-by-user", "data": { "userId": "string", "feedId": number } }

Note:

  • This event is broadcast only to users in the call
  • Contains the user ID and feed ID of who lowered their hand
  • Can be used to remove the hand raised indicator in the UI
  • Triggered when a user sends a lower-hand message
  • feedId identifies which specific media handle lowered the hand

Error Handling

Error responses and validation failures

ERROR
error
Error message from server
▼
Error Schema:
{ "type": "error", "error": "string" // Human-readable error message }
Common Error Types:

Validation Errors:

  • Missing fields: "Missing required field: fieldName"
  • Type mismatch: "Field 'fieldName' expected string, but received number"
  • String length: "Field 'fieldName' must be at least X characters long"
  • Invalid UUID: "Field 'fieldName' must be a valid UUID"
  • JSON parsing: "Invalid JSON format"

Connection Errors:

  • "User not authenticated"
  • "User not found in this room"
  • "Invalid credentials"
  • "Connection failed"

Message Errors:

  • "Unknown message type"
  • "User not found"
  • "Socket error occurred"
